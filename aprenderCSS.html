Reset

São arquivos padrozinados que subscreve todo esse estilo padrão dos browsers.
É importante usar para ter uma base sólida para o desenvolvimento. Para usa-lo devemos declarar no link no corpo HTML.
EXs: Eric Meyer: arquivo bem pequeno; deixa todos os elementos com mesma aparência e tamanho de fonte;
Normalize: arquivo um pouco maior, mas com ajustes mais finos, já define fontes e tamanhos padrão para alguns elementos e corrige diversas inconsistências entre navegadores;
YUI (Yahoo!): arquivo bem pequeno, com um efeito bem parecido com o do Eric Meyer, mas com alguns ajustes mais específicos.

IMPORTANTE: CONFLITOS:

Onde declarar o reset, se vc declarar estilo diferente para a mesma tag?
O navegador le as regras na sequência, ficando com a ultima declaração. Também importa a ordem de especificação

Com Imagem

Se estiverem representando visualmente já sabemos: CSS nele. Com isso no html escrevemos o nome e no css usamos background-image.
Para eliminar o texto que esta por cima da imagem usamos text-ident: -1000px isso jogara para fora da sua pagina "escondendo"
(regula o recuo da primeira linha de um paragrafo)
Chamamos essa "solução" de image replacement. Por outro lado, usar o font-size com valor zero não é bem interpretado por todos os navegadores. Alguns simplesmente ignoram uma declaração desse tipo; outros, mesmo com o tamanho zerado, mostram o texto numa fonte pequena. Ou seja, há um problema de compatibilidade com essa técnica, que a propriedade text-indent não tem.

Positions

Relative: conseguimos mudar a posição do elemento de acordo com a posição original dele.
Absolute: podemos colocar um elemento onde quisermos na página. Nem sempre ele é relativo a página, ele pode estar relativo ao primeiro elemento pai definido.
Fixed: nos permite deixar um elemento no lugar que quisermos.

Pixels

Utilizamos a porcentagem para termos um controle do tamanho da fonte da página com base no tamanho padrão da fonte do navegador, também podemos definir larguras.
s A % esta relacionada ao container, o pai do elemento, a relação é sempre entre as larguras. Se um elemento é definido em pixels, não importa quanto mudemos a configuração ele manterá a configuração.

Medida em REM - é uma medida muito boa para quando quisermos que os elementos sigam o padrão do texto principal.
Tem como padrão a fonte do navegador.

A medida ch - é boa porque não importa muito a fonte que você está utilizando, ela se adequará muito bem.
Tem como padrão a fonte do elemento pai.

A medida em - indica o tamanho da fonte do elemento, não do navegador.
Tem como base a largura do caractere zero da fonte usada. É muito mais acessível e flexível que o pixel;
Para Evitar cálculos usamos o REM que deixará o valor do contexto o mesmo sempre(body)
